## Usage Guide

### For Architect Mode Agents

1. Review this index before making similar architectural decisions
2. Reference previous context and appetite constraints that influenced choices
3. Document new decisions with clear rationale and trade-off analysis
4. Update this index when ADRs are created or updated

### For Understanding Context

- Decisions explain why specific approaches were chosen within appetite boundaries
- Trade-off analysis helps understand constraints and limitations
- Previous decisions inform future appetite allocation and planning

### Decision Structure

Each decision record includes:

- **Decision**: Clear statement of what was decided
- **Context**: Appetite constraints and requirements that influenced the choice
- **Rationale**: Why this approach was selected over alternatives
- **Consequences**: Expected outcomes and trade-offs
- **Status**: Proposed, Accepted, Superseded, or Deprecated
